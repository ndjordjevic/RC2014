## Landing page — CP/M (Wikipedia)

CP/M (Control Program for Microcomputers) is a disk operating system created in 1974 by Gary Kildall at Digital Research, Inc. Originally for Intel 8080 microcomputers, it became the de facto standard and dominant OS for microcomputers from the late 1970s to mid-1980s.

Architecture: three core components — BIOS (hardware drivers), BDOS (file system and services), and CCP (command-line interpreter). Programs ran as transient `.COM` files loaded into memory; the abstraction layer enabled rapid porting across hardware.

Key versions: CP/M 2.2 (most widely deployed); CP/M 3.0 / "Plus" (1983, bank-switched memory beyond the 64KB limit). Ran on Intel 8080/8085 and Zilog Z80; 16-bit variants CP/M-86 and CP/M-68K followed. Z80 systems dominated due to backward compatibility.

Significance: standardized I/O and file management reduced porting costs and enabled a large software library; displaced by MS-DOS after the 1981 IBM PC. Still active in retrocomputing (e.g. RC2014); source now available under open-source licenses.
