- Description: "Official RC2014 repository for hardware schematics / software and more!"

Top-level structure:
- `Hardware/` — schematics and mechanical files (512kROMRAM, APU, MC68C50.pdf, RC2014 Mini Dimensions.svg, Port Cover/Rear Panel SCAD/SVG, SDDump, Sound, rc2014-82c55-ide.pdf).
- `ROMs/` — ROM build trees and Makefiles (CPM-IDE, Factory, MSBASIC, SIO2, TinyBasicPlus, pigfx, snake, helloworld, init/init_ram; buildrules.mk, config.mk).
- `CPM/` — CP/M disk and Compact Flash images plus uploaders (Python and binary), MonitorLoad.z80.
- `BASIC-Programs/` — example programs (Mandelbrot, Super Star Trek, hexload, primes, Mastermind) and reference PDFs (LIB-80, LINK-80, MS BASIC Compiler 1980, NASCOM Basic Manual).
- `Emulators/` — Rocket2014, z80-machine.
- Third-party projects (pigfx, emulators) are git submodules; initialise with `git submodule update --init`. For users without an EEPROM burner, the `hexload` BASIC program uploads work over serial.

## Repo wiki — Using Z88DK with RC2014 — https://github.com/RC2014Z80/RC2014/wiki/Using-Z88DK

Guide to cross-developing C for the RC2014 with z88dk ("a complete development toolkit for the 8080, 8085, gbz80, z80, z180 (ez80), z80n and Rabbit processors"). Build z88dk from source (`git clone`, apt deps, `git submodule update --init --recursive`, `export BUILD_SDCC=1`, `./build.sh`), set PATH and ZCCCFG. Two C compilers: z88dk-sccz80 (small-C derivative) and z88dk-zsdcc (patched optimizing SDCC). Classic vs new (ROM-able) libraries; floating point via math48/32/16; `__sfr` for I/O. The `+rc2014` target has six subtypes: ACIA, SIO (ROM code), BASIC (RAM at 0x9000), CPM (0x0100), HBIOS/RomWBW (0x0100), and 8085 variants. Example: `zcc +rc2014 -subtype=basic -SO3 --max-allocs-per-node200000 sudoku.c -o sudoku -create-app`.

## Repo folder — Factory ROMs — https://github.com/RC2014Z80/RC2014/tree/master/ROMs/Factory

The official factory ROM images (.BIN/.HEX) as supplied with RC2014 products, plus `Pi Serial Terminal` and `RomWBW` subfolders. The README decodes the 8-digit ROM label: each digit (left→right) names the firmware in one 8K bank from 0x0000 to 0xD000, selected by the A13/A14/A15 jumpers. Digit map: 0 = empty/user; R = MS-BASIC (32k RAM, 68B50 ACIA); K = MS-BASIC (56k RAM, ACIA); 1 = CP/M Monitor (pageable ROM, 64k RAM, ACIA, CF at 0x10); 2 = MS-BASIC (32k, SIO/2); 4 = MS-BASIC (56k, SIO/2); 6 = CP/M Monitor (SIO/2, CF at 0x10); 88 = Small Computer Monitor (pageable ROM, 64k, SIO/2 or ACIA, with BASIC+CP/M boot); 9 = Small Computer Monitor (any ROM/RAM/UART). Files with "plus" suffix target the Micro Plus / Classic II Plus (39SF010 128k ROM).
